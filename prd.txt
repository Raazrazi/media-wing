# PRODUCT REQUIREMENTS DOCUMENT (PRD)

## Project Title

Students' Union Media Request & Poster Registration Portal

## Version

1.0

## Owner

Students' Union Media Wing

## Purpose

To establish a centralized digital system through which all committees, class unions, and subcommittees can submit poster and publicity requests to the Media Wing.

The system will enforce submission deadlines, streamline communication, maintain records, and improve planning efficiency.

---

# Problem Statement

Currently, poster requests are submitted through informal channels, resulting in:

* Late submissions
* Missing information
* Difficulty tracking requests
* Lack of accountability
* Increased workload for the Media Wing

The portal aims to solve these issues through structured digital registration and automated validation.

---

# Objectives

1. Centralize all poster requests.
2. Enforce a mandatory 48-hour submission deadline.
3. Maintain a permanent request database.
4. Allow Media Wing administrators to review requests efficiently.
5. Improve communication between organizers and the Media Wing.
6. Generate reports for auditing and union documentation.

# Functional Requirements

## FR-01 Program Registration Form

Required Fields:

* Program Name
* Program Title
* Program Date
* Program Time
* Venue
* Organizing Committee
* Program Category
* Detailed Description
* Poster Requirements
* Contact Person
* Phone Number

All fields are mandatory unless specified otherwise.

---

## FR-02 Unique Registration ID

System shall generate IDs automatically.

Format:

REQ-YYYY-001

Example:

REQ-2026-001

---

## FR-03 48-Hour Deadline Validation

System shall compare:

Current Time

with

Program Date & Time

Rule:

Submission must be at least 48 hours before the event.

If less than 48 hours remain:

Registration shall be blocked.

---

## FR-04 Warning Message

Display:

WARNING

Poster registration closes strictly 48 hours before the scheduled program time.

Requests submitted after this deadline will not be accepted and may be automatically canceled by the Media Wing.

Please contact the Media Chairman for special approval.

---

## FR-05 Database Storage

Every successful submission shall store:

* Timestamp
* Registration ID
* Request Details
* Status

Database:

Google Sheets (Phase 1)

Future:

Firebase / PostgreSQL

---

## FR-06 Submission Confirmation

After successful registration:
